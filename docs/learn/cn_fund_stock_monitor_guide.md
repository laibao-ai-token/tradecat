# TradeCat 国内页选股/基金监控指南（新手版）

> 适用页面：`TUI -> market_cn`  
> 目标：用同一页做“候选筛选 + 盘中监控 + 风险控制”

## 1) 先说清楚：这个页面现在到底是什么

- 当前 `market_cn` 本质是 **A 股行情监控页**，不是独立的“基金研究页”。
- 你可以监控：
  - A 股个股（如 `SH600519`）
  - 场内 ETF/LOF（如 `SH510300`、`SZ159915`）
- 你不能直接监控：
  - 场外公募基金净值（按日更新那类 NAV）

结论：你现在用于“选股/选 ETF”是可行的，但它更偏 **盘中交易监控**，不是基金基本面分析工具。

## 2) 页面里的数据分别来自哪里

- 行情（价格/涨跌/成交量）：`services-preview/tui-service/src/quote.py`，CN 默认走 Tencent 行情。
- 价格曲线（左上 Price Curve）：基于实时行情构建 5s 聚合曲线。
- Signal Radar / Rule Signals（右上/左下）：
  - 来自 `libs/database/services/signal-service/signal_history.db` 的 `signal_history` 表。
  - 如果信号库里没有对应 CN 代码，就会看到 `no recent rule signals`（这是正常现象，不是页面坏了）。
- micro 的 `bias/score`：来自 `services-preview/tui-service/src/micro.py` 的轻量打分（EMA/RSI/ROC/VWAP/whale），**不是 129 规则本体**。

## 3) 新手可直接照抄的使用流程（选股/选 ETF）

1. 先建候选池（10~30 个）
   - 用 `--cn-symbols` 或在页面里 `+`/`-` 增删。
   - ETF 代码建议始终带前缀：`SH510300,SZ159915`（更稳，不易误判）。
2. 第一轮筛选：流动性
   - 先看 `vol`，剔除明显冷门、盘口薄的标的。
3. 第二轮筛选：方向一致性
   - 看 `chg/pct` 与价格曲线是否同向（上涨+曲线抬升，或下跌+曲线走弱）。
4. 第三轮筛选：信号确认
   - 看右上 `Signal Radar` 是否有近 15 分钟、强度 >= 60 的信号。
5. 最后执行：仓位与止损
   - 单票先小仓位，先用固定止损（例如 1~2%）控制回撤。

## 4) 你最需要注意的 8 个坑

1. **别把 micro score 当“必涨指标”**：它是短线统计信号，不是确定性预测。
2. **没信号不等于没机会**：可能只是 `signal_history` 没覆盖该票。
3. **ETF 输入请带交易所前缀**：`SH/SZ + 6 位代码`。
4. **开盘前后噪声最大**：9:30 附近波动快，先观察再动作更稳。
5. **午间休市要看 age**：`age` 变大是正常，不是断线。
6. **只看涨跌不够**：必须同时看成交量与曲线形态。
7. **不要满仓单票**：建议分批建仓，避免一次判断失误带来大回撤。
8. **先做“候选池管理”再做“买卖”**：流程顺序会显著降低随机交易。

## 5) 推荐启动方式（国内页）

```bash
cd services-preview/tui-service
./scripts/start.sh run --view market_cn --cn-symbols SH600519,SZ000001,SH510300,SZ159915 --quote-refresh 1
```

页面内常用键：
- `2`：切到国内页
- `[` / `]`：切换标的
- `+` / `-`：增删候选
- `q`：退出

## 6) 你下一步最值得做的两件事

- 先固定一套“候选池 + 入场/止损”纪律，连续观察 5~10 个交易日，避免频繁改规则。
- 如果你希望“国内页专用选股”更强，下一步建议做：
  1) 候选池排序（按成交额/波动率/信号强度）  
  2) 告警阈值（例如强度>=70 且 5 分钟内出现 2 次同向信号）
