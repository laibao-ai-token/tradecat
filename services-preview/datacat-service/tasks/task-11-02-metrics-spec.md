# 任务 11-02：验收指标与阈值定义

## 目标

- 定义字段/行数/日志一致性指标。

## 验收指标（强制）

1) 字段一致性
   - 列名一致
   - 数据类型一致（数值/时间/布尔）
   - 精度一致（小数位不丢失）

2) 行数一致性
   - 实时窗口：允许误差 ≤ 1%
   - 回填窗口：允许误差 ≤ 0.5%

3) 时间对齐一致性
   - K线：bucket_ts 对齐到 1m
   - 指标：create_time 对齐到 5m
   - 最大偏移允许：±1 个周期

4) 来源一致性
   - ZIP 数据 source=binance_zip
   - REST 数据 source=binance_rest/ccxt_gap
   - WS 数据 source=binance_ws

## 阈值说明

- 误差超过阈值即判为失败。
- 允许小误差以适配实时写入延迟与并发写入。

## 验收

- 阈值明确。
